- **Verification:** 0 zaraz references remaining (was 22)
- **Result:** BSMA loads clean, no Zaraz conflict

## 18:57 onwards — BSMA Debug Marathon: "toggleCall is not defined"

### Root Cause Chain

**Bug 1: `Invalid or unexpected token`**
- Raw uncomputed em-dash line (`──────────────────────────────────`) sitting bare between `}` and `const callStart = Date.now()` — a copy-paste separator artifact
- Fix: Wrapped in `/** */` comment

**Bug 2: `const callStart` redeclared** — two `const callStart` declarations in the same script (one at line ~2126 from the visual fix, another at ~2144 from original code)
- Fix: Removed the duplicate at line ~2144

**Bug 3: `toggleCall is not defined`** — All browser code was wrapped in an IIFE `(function(){...})();` to protect against Zaraz duplication, but HTML `onclick="toggleCall()"` looks in the **global scope**
- Fix: Exposed all 4 onclick functions (`toggleCall`, `toggleLang`, `quickSend`, `quickAction`) on `window` object at the end of the IIFE

**Bug 4 (side-effect): `window.bsmaBookAppt = bsmaBookAppt` causing ReferenceError** — `bsmaBookAppt` doesn't exist as a standalone function (it's defined dynamically later)
- Fix: Removed all `window.xxx = xxx` assignments for dynamically-defined functions

**Bug 5: Server-side `\n` in template literal** — Cloudflare edge processing converts `\n` to real newlines inside JS single-quoted strings
- Fix: Split into concatenated strings to avoid any `\n` in template literals

### Zaraz Persistence
- **Zaraz cannot be fully removed by editing the Worker HTML** — Cloudflare still injects Zaraz scripts at the **edge level** (not in Worker source)
- **Mitigation:** Duplicating the entire `<script>` content for each inline script was causing `const` redeclaration. Fix: Wrap all browser JS in IIFEs `(function(){...})();` so duplicate invocations create new lexical scopes

3. `onend` auto-restart was also firing
4. This created a cycle: greeting fires → recognition stops → recognition restarts → but user's speech was lost in between

### Voice Flow v3.0 — Complete Rewrite (295 lines replaced)

**Core changes:**
- `continuous: false` — single utterance per cycle, clean restart
- `interimResults: false` — final results only, no interim noise
- **Recognition never stopped** during AI processing — removed all `recognition.stop()` calls
- Clean restart model: ONLY through `onend` handler, never from `finally` or other paths
- Removed `isDifferent` / `previousTranscript` dedup logic — caused more issues than it solved

**New architecture:**
1. User speaks → recognition captures final text → waits 800ms silence → sends to AI
2. AI processing happens WITHOUT stopping recognition
3. While speaking response, recognition is paused via voiceStoppedByUser
4. After speaking done → restart recognition naturally via onend
5. No race conditions, no double-start, no lost speech

**New `detectAndRunTool(userMsg)` — Proactive intent detection:**
- **Emergency** 🚑: Detects chest pain, shortness of breath, bleeding → redirect to ER immediately
- **Appointment** 📅: Fetches real available slots from CRM
- **Insurance** 🏥: Checks NPHIES eligibility, asks for national ID if needed
- **Claims** 💰: Fetches live NPHIES GSS/PA data
- **Patient lookup** 👤: Searches by national ID via Oracle eligibility bridge
