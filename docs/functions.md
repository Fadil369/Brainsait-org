# ⚙️ BrainSAIT Pipeline Functions Reference

> **Total Functions:** 3 global filter functions  
> **Type:** All filters (input + output pipeline)  
> **Scope:** Global (apply to all agents and channels)

---

## 🔄 Pipeline Architecture

All messages flow through the three global filter functions in priority order **before reaching the LLM** (inlet) and **after the LLM responds** (outlet):

```
User Message
     ↓
[1] PHI Guardian (priority 10) — inlet: mask PHI
     ↓
[2] Compliance Auditor (priority 5) — inlet: flag issues
     ↓
LLM / Agent Processing
     ↓
[2] Compliance Auditor — outlet: append audit trail
     ↓
[3] Bilingual Formatter (priority 15) — outlet: format AR/EN
     ↓
Final Response to User
```

---

## 🛡️ PHI Guardian

| Property | Value |
|----------|-------|
| **Function ID** | `phi_guardian` |
| **Display Name** | PHI Guardian |
| **Type** | `filter` |
| **Scope** | Global (all agents) |
| **Priority** | 10 |
| **Is Global** | ✅ Yes |

### What It Does

Automatically detects and masks Protected Health Information (PHI) before it reaches the LLM. Prevents inadvertent training on or logging of sensitive patient identifiers.

#### Runs On
- **Inlet (input):** ✅ Yes — processes every user message before LLM sees it
- **Outlet (output):** ❌ No — PHI masking happens before processing only

#### PHI Patterns Detected & Masked
| PHI Type | Pattern | Replacement |
|----------|---------|-------------|
| Saudi National ID | 10-digit number starting with 1 or 2 | `[NATIONAL-ID-REDACTED]` |
| Iqama (Residency) | 10-digit number starting with 2 | `[IQAMA-REDACTED]` |
| Phone Numbers | Saudi (+966) and international formats | `[PHONE-REDACTED]` |
| Email Addresses | Standard email format | `[EMAIL-REDACTED]` |
| NPHIES Patient ID | NPHIES-prefixed identifiers | `[NPHIES-ID-REDACTED]` |

#### Implementation Details
```python
class Filter:
    class Valves:
        priority: int = 10  # runs second in pipeline

    def inlet(self, body, user):
        # Regex-based PHI detection and masking
        # Applied to all user messages before LLM processing
        ...

    # No outlet method — PHI masking is input-only
```

#### Configuration (Valves)
| Valve | Default | Description |
|-------|---------|-------------|
| `priority` | `10` | Pipeline execution order |

## 📋 Compliance Auditor

| Property | Value |
|----------|-------|
| **Function ID** | `compliance_auditor` |
| **Display Name** | Compliance Auditor |
| **Type** | `filter` |
| **Scope** | Global (all agents) |
| **Priority** | 5 |
| **Is Global** | ✅ Yes |

### What It Does

Appends a HIPAA/NPHIES compliance audit trail to every response generated by LINC agents. Ensures all AI-generated medical content is traceable, timestamped, and flagged with appropriate compliance warnings.

#### Runs On
- **Inlet (input):** ✅ Yes — audits incoming messages for compliance triggers
- **Outlet (output):** ✅ Yes — appends audit trail block to every response

#### Audit Trail Format

Every LINC agent response includes an appended audit block:

```
---
🔒 COMPLIANCE AUDIT TRAIL
━━━━━━━━━━━━━━━━━━━━━━━━
Audit ID: [uuid]
Timestamp: [ISO 8601]
Agent: [agent_id]
User: [user_id]
HIPAA Compliant: ✅
NPHIES Aligned: ✅
PHI Detected: [yes/no]
Action Logged: HIPAA_AUDIT_LOGGER
```

#### Compliance Checks
- HIPAA minimum necessary standard validation
- NPHIES data sharing requirement compliance
- PDPL (Saudi) personal data handling flags
- PHI access justification verification
- Clinical decision support disclaimer injection

#### Configuration (Valves)
| Valve | Default | Description |
|-------|---------|-------------|
| `priority` | `5` | Runs first in pipeline (lowest number = first) |
| `show_audit_trail` | `true` | Show/hide audit block in response |
| `hipaa_mode` | `strict` | Compliance strictness level |

## 🌍 Bilingual Formatter

| Property | Value |
|----------|-------|
| **Function ID** | `bilingual_formatter` |
| **Display Name** | Bilingual Formatter |
| **Type** | `filter` |
| **Scope** | Global (all agents) |
| **Priority** | 15 |
| **Is Global** | ✅ Yes |

### What It Does

Ensures responses are clearly separated into English and Arabic sections for maximum readability across Saudi healthcare teams. Normalizes bilingual output format across all agents — even agents that don't explicitly generate bilingual responses.

#### Runs On
- **Inlet (input):** ❌ No — formatting is applied to outputs only
- **Outlet (output):** ✅ Yes — reformats every response into structured bilingual layout

#### Output Format Template

When Arabic content is detected in a response, the formatter restructures it as:

```
## 🇬🇧 English
[English content block]

---

## 🇸🇦 العربية
[Arabic content block]
```

#### Detection Logic
- Scans response for Arabic Unicode characters (U+0600–U+06FF)
- If Arabic detected: applies bilingual separation
- If English-only: passes through unchanged
- Handles mixed inline content: separates into clean blocks
- Preserves markdown formatting, code blocks, and tables

#### Implementation Details
```python
import re

class Filter:
    class Valves:
        priority: int = 15  # runs last in output pipeline

    def outlet(self, body, user):
        # Detect Arabic script in response
        # Split and reorganize into EN/AR blocks
        # Apply section headers with flag emojis
        ...
```

#### Configuration (Valves)
| Valve | Default | Description |
|-------|---------|-------------|
| `priority` | `15` | Runs last in output pipeline |
| `always_bilingual` | `false` | Force bilingual format even for English-only responses |
| `arabic_header` | `🇸🇦 العربية` | Arabic section header |
| `english_header` | `🇬🇧 English` | English section header |

## 📊 Function Pipeline Summary

| Order | Function | Priority | Inlet | Outlet | Purpose |
|-------|----------|---------|-------|--------|---------|
| 1st | Compliance Auditor | 5 | ✅ | ✅ | Audit flags + trail |
| 2nd | PHI Guardian | 10 | ✅ | ❌ | Mask PHI before LLM |
| 3rd | Bilingual Formatter | 15 | ❌ | ✅ | Format AR/EN output |

> **Note:** Lower priority number = runs first. The Compliance Auditor runs before PHI Guardian to capture the audit event, then PHI Guardian masks sensitive data.
